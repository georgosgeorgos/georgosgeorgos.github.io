Mathematics

Inline math works naturally: the loss function \mathcal{L}(\theta) is minimized via gradient descent with learning rate \alpha. Gradient descent is the workhorse of modern optimization. See Ruder (2016) for a comprehensive overview of variants.

Display math renders centered:

\nabla_\theta \mathcal{L} = \mathbb{E}_{x \sim p(x)} \left[ \nabla_\theta \log p_\theta(x) \right]

The evidence lower bound (ELBO) is: The ELBO was introduced by Jordan et al. (1999) and forms the basis of variational inference.

\text{ELBO} = \mathbb{E}_{q(z|x)} \left[ \log p(x|z) \right] - D_{\text{KL}}\left( q(z|x) \| p(z) \right)

Code

Here is a Python function with syntax highlighting:

def kl_divergence(mu, log_var):
    """Compute KL divergence for a Gaussian posterior."""
    return -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())

And inline code like torch.optim.Adam(lr=1e-3) renders with a subtle background. Adam combines momentum and adaptive learning rates. See Kingma & Ba (2015) for the original paper.
